# 云端更新标准流程

> 适用服务器：`47.114.111.216`
> 目标：以后每次只更新功能代码时，按本文执行，避免误改已有数据。

## 一、当前云端真实路径

- Git 仓库目录：`/www/wwwroot/xuetoushiguanli`
- Nginx 实际前端目录：`/opt/hemodialysis/frontend/dist`
- PM2 后端实际目录：`/opt/hemodialysis/backend`
- 后端环境变量文件：`/opt/hemodialysis/backend/.env`
- 后端服务名：`hd-backend`
- 后端健康检查：`http://localhost:3080/health`
- 备份目录：`/www/backup/xuetoushiguanli`

注意：宝塔网站列表可能显示 `/www/wwwroot/xuetoushiguanli/frontend/dist`，但 Nginx 实际服务的是 `/opt/hemodialysis/frontend/dist`。更新前端后必须同步到 `/opt/hemodialysis/frontend/dist`。

## 二、本地更新前

在本地完成开发后，先检查：

```bash
cd E:/xuetoushiguanli
```

前端检查：

```bash
cd frontend
npm run lint
npm run build
```

后端关键加载检查：

```bash
cd ../backend
node -e "require('./src/routes/patients'); require('./src/services/AiAnalysisService'); console.log('backend ok')"
```

提交并推送：

```bash
cd ..
git status --short
git add <本次需要提交的文件>
git commit -m "fix: 本次更新说明"
git push origin master
```

不要提交：

- `backend/.env`
- `backend/logs`
- `backend/uploads`
- 服务器密码、宝塔账号、数据库备份
- 临时截图、临时脚本、个人文档

## 三、云端更新命令

可以从本机执行，也可以 SSH 登录服务器后执行。

### 1. 拉取最新代码

```bash
cd /www/wwwroot/xuetoushiguanli
git status --short
git stash push -u -m "cloud-local-before-update-$(date +%Y%m%d-%H%M%S)"
git pull origin master
```

### 2. 创建备份

```bash
BACKUP_DIR=/www/backup/xuetoushiguanli/$(date +%Y%m%d-%H%M%S)
mkdir -p "$BACKUP_DIR"

tar -czf "$BACKUP_DIR/www-project-before-update.tar.gz" \
  --exclude='backend/node_modules' \
  --exclude='frontend/node_modules' \
  --exclude='frontend/dist' \
  --exclude='backend/logs' \
  --exclude='backend/uploads' \
  -C /www/wwwroot xuetoushiguanli

tar -czf "$BACKUP_DIR/opt-backend-before-update.tar.gz" \
  -C /opt/hemodialysis/backend src migrations package.json package-lock.json .env.example

tar -czf "$BACKUP_DIR/opt-frontend-dist-before-update.tar.gz" \
  -C /opt/hemodialysis/frontend dist

cp /opt/hemodialysis/backend/.env "$BACKUP_DIR/backend.env.bak"
pg_dump -U hd_app -h localhost hemodialysis_db > "$BACKUP_DIR/hemodialysis_db.sql"
```

### 3. 同步后端到 PM2 实际目录

```bash
rsync -a /www/wwwroot/xuetoushiguanli/backend/src/ /opt/hemodialysis/backend/src/
rsync -a /www/wwwroot/xuetoushiguanli/backend/migrations/ /opt/hemodialysis/backend/migrations/
cp /www/wwwroot/xuetoushiguanli/backend/package.json /opt/hemodialysis/backend/package.json
cp /www/wwwroot/xuetoushiguanli/backend/package-lock.json /opt/hemodialysis/backend/package-lock.json
cp /www/wwwroot/xuetoushiguanli/backend/.env.example /opt/hemodialysis/backend/.env.example

cd /opt/hemodialysis/backend
npm install --production
```

### 4. 数据库迁移规则

默认不要运行全部迁移。

只有当本次更新明确新增必要迁移文件，并确认不会清空数据时，才单独执行对应迁移，例如：

```bash
cd /opt/hemodialysis/backend
psql -U hd_app -h localhost -d hemodialysis_db -f migrations/060_patients_identifier_status_hospitalized.sql
```

如果不确定，先停止，不要执行迁移。

### 5. 构建并同步前端

```bash
cd /www/wwwroot/xuetoushiguanli/frontend
npm install
npx tsc -b
npx vite build --emptyOutDir=false

mkdir -p /opt/hemodialysis/frontend/dist
rsync -a --delete --exclude='.user.ini' \
  /www/wwwroot/xuetoushiguanli/frontend/dist/ \
  /opt/hemodialysis/frontend/dist/
```

### 6. 重启与检查

```bash
pm2 restart hd-backend
nginx -t
systemctl reload nginx
curl -sS -m 10 http://localhost:3080/health
```

正常结果应包含：

```json
{"status":"ok"}
```

## 五、回滚思路

如前端异常：

```bash
BACKUP_DIR=<本次备份目录>
rm -rf /opt/hemodialysis/frontend/dist
tar -xzf "$BACKUP_DIR/opt-frontend-dist-before-update.tar.gz" -C /opt/hemodialysis/frontend
nginx -t && systemctl reload nginx
```

如后端异常：

```bash
BACKUP_DIR=<本次备份目录>
tar -xzf "$BACKUP_DIR/opt-backend-before-update.tar.gz" -C /opt/hemodialysis/backend
cp "$BACKUP_DIR/backend.env.bak" /opt/hemodialysis/backend/.env
pm2 restart hd-backend
curl -sS -m 10 http://localhost:3080/health
```

数据库备份只在确认数据库被误操作时才恢复，不要随意恢复数据库。
