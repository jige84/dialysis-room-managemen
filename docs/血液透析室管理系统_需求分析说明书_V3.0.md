## 一、项目背景与目标

### 1.1 现状与痛点

目前透析室在信息管理方面存在以下问题：

- 透析记录仍使用纸质表单，护士书写耗时，字迹难以辨认，查阅历史记录不便
- 患者检验数据分散，无法快速查看趋势，Kt/V、URR、β₂微球蛋白清除率等指标需手动计算
- 传染病筛查到期提醒依靠人工核对，存在漏提醒风险
- 月度质控报表统计耗费大量人力，**向质控中心上报的五项指标（护患比、凝血率、漏血率、穿刺损伤率、CRBSI）** 需每月手工汇总，容易出现统计误差
- AVF患者登记表、临时管/长期管统计表、导管感染监测、溶栓记录、CQI记录等均为独立纸质表单，无法联动查询
- 护患比统计依赖每日手工登记累加，无法实时查看
- 体外循环凝血分级（Ⅰ/Ⅱ/Ⅲ级）每日统计依赖人工填表，月度汇总费时
- 医用耗材出库管理靠手工台账，容易差错

### 1.2 建设目标

1. 替代全部现有纸质台账，实现患者全生命周期数据数字化管理
2. 透析记录一次录入，自动计算 Kt/V、URR、超滤量等关键指标
3. **一键生成质控中心上报报表**（5项质控指标），数据自动从日常录入中汇总
4. 血管通路全链路管理：AVF/临时管/长期管台账、导管感染月度监测、CVC感染高危评估、溶栓记录联动
5. 实验室数据结构化存储，自动标记异常值和危急值，按规程要求频率提示复查
6. 严格落实传染病防控，满足规程第3章感染控制要求
7. 通过 AI 大模型辅助分析患者数据，提供参考性风险提示
8. 保障患者数据安全，满足医疗信息系统合规要求
9. 医用耗材自动出库管理

### 1.3 系统边界（本期不做）

> ⚠️ **边界说明**

- 不对接医院 HIS 系统（列入后期规划）
- 不做透析机实时数据采集（数据手动录入）
- 不做财务计费功能
- 不做移动端 App（第一版只做网页版，手机浏览器可访问）
- 不做透析用水处理设备在线监控（水质检测结果人工录入）

#### 3.2.6 Kt/V 和 URR 自动计算

**Kt/V（Daugirdas II 公式）：**

```
spKt/V = -ln(R - 0.008×t) + (4 - 3.5×R) × (UF/W)
R = 透析后BUN ÷ 透析前BUN
t = 实际透析时长（小时）
UF = 超滤量（L）
W = 透析后体重（kg）
```

**URR（尿素下降率）：** `URR(%) = (1 - R) × 100%`

**判断标准：** spKt/V ≥ 1.2 且 URR ≥ 65% → 透析充分；任一不达标 → 标记并提醒主管医生。

| AVF 使用率 | AVF在透患者数/全部在透患者数×100% | 尽可能高 |
| AVF 长期生存率 | 同一AVF持续使用>2年的患者比例 | — |
| CVC 使用率 | CVC（临时+长期）患者数/全部在透患者数×100% | — |

**感染质控：**

| 指标 | 计算逻辑 |
|------|---------|
| 新入患者传染病检验完成率 | 完成4项检验的新入患者数/新入患者总数×100% |
| 维持透析患者传染病定时检验完成率 | 每6个月按时完成检验的患者数/应检患者数×100% |
| 年度新发乙肝/丙肝发病率 | 新发阳转患者数/年平均在透患者数×100% |

**并发症质控：**

| 指标 | 计算逻辑 |
|------|---------|
| **高血压控制率** | 透析前血压达标患者比例（<60岁:<140/90；≥60岁:<160/90） |
| **肾性贫血控制率** | Hb≥110 g/L 的患者比例 |
| **CKD-MBD 指标控制率** | 钙2.10-2.50且磷1.13-1.78且iPTH 2-9×ULN三项同时达标比例 |
| **血清白蛋白控制率** | ALB≥35 g/L 的患者比例 |
| **Kt/V 达标率** | spKt/V≥1.2 的患者比例 |
| **URR 达标率** | URR≥65% 的患者比例 |
| **透析间期体重增长控制率** | 透析间期体重增长<5%干体重的患者比例 |

### 3.14 AI 大模型辅助分析【P2 增值功能】

> ⚠️ **合规提醒** AI分析结果必须标注：「本内容由AI生成，仅供医护人员参考，**不构成医疗诊断建议**」。已与医院法务确认合规。

**拟实现功能：**

- **单患者透析趋势解读**：输入近3个月数据，AI总结Kt/V/血压/体重增长趋势，结合最新指南提示需关注的问题
- **检验结果综合分析**：对照规程目标范围解读最新生化指标
- **Kt/V不达标原因辅助分析**：结合透析参数辅助排查不达标原因
- **CVC感染高危评分解读**：AI解读评分原因，提出针对性护理建议
- **自然语言查询**：如"张三最近三个月Kt/V趋势"、"上月哪些患者超滤量超标"
