## 五、修改定位规则（快速索引指南）

### 规则 1：前后端对应关系

修改某个业务功能时，通常需要同时改动：
```
后端路由  →  backend/src/routes/{module}.js
后端服务  →  backend/src/services/{Service}.js
前端API   →  frontend/src/api/{module}.ts
前端页面  →  frontend/src/pages/{Module}/
```

### 规则 2：新增功能模块检查清单

1. 后端：`routes/` 新增路由文件 → `server.js` 注册路由
2. 后端：`migrations/` 新增迁移文件 → 运行 `runMigrations.js`
3. 后端：`middleware/menuPermission.js` 新增模块权限映射
4. 前端：`api/` 新增接口文件
5. 前端：`pages/` 新增页面目录
6. 前端：`App.tsx` 新增路由配置
7. 前端：`constants/sidebarModules.ts` 新增菜单项
8. 前端：`utils/menuAccess.ts` 新增角色权限映射

### 规则 3：常见修改场景速查

| 修改需求 | 定位文件 |
|----------|----------|
| 修改登录逻辑 | `backend/src/routes/auth.js` + `frontend/src/api/auth.ts` + `frontend/src/pages/Login/` |
