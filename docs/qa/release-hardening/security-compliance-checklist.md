# 上线安全与合规验收清单

> 对应计划步骤：8（安全与合规专项验收）  
> 验收日期：______  
> 验收人：______  
> 环境：预发布 / 生产演练

## 1. 传输与访问安全

- [ ] 全站 HTTPS 生效，HTTP 自动跳转到 HTTPS
- [ ] API 网关/反向代理对外仅暴露必要端口
- [ ] 登录与全局速率限制生效（含返回码 429）

## 2. 认证与会话安全

- [ ] JWT 签发、校验、过期策略符合现网约束
- [ ] 登出后 token 吊销生效（黑名单命中返回 401）
- [ ] 连续输错密码 5 次锁定 30 分钟策略生效

## 3. RBAC 权限安全（5角色抽测）

- [ ] `admin` 关键管理接口可访问
- [ ] `head_nurse` 具备护士扩展权限但无 admin-only 能力
- [ ] `doctor` 可医生域操作但不可越权管理
- [ ] `nurse` 仅当班写入，不可越权修改历史
- [ ] `qc/quality` 保持只读边界与兼容映射

## 4. 数据与隐私合规

- [ ] 身份证/手机号加密存储（数据库无明文）
- [ ] API 响应不返回敏感明文字段
- [ ] 日志脱敏生效（禁止身份证/手机号/token/password 原文）

## 5. 审计与可追溯

- [ ] 核心写操作可在审计日志追溯（人、时间、对象、内容）
- [ ] 审计记录不可被普通角色删除/篡改
- [ ] 关键失败请求可定位到请求链路与错误类型

## 6. AI 与扩展模块（如启用）

- [ ] AI 接口权限限制仅允许授权角色访问
- [ ] AI 输出包含免责声明
- [ ] 知识库写入遵守“用户确认后保存”策略

## 7. 结论

- [ ] 通过（允许进入上线切换）
- [ ] 不通过（阻断上线，附整改项）
