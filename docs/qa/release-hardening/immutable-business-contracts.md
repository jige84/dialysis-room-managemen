# 不可变业务契约清单（Immutable Business Contracts）

> 更新时间：2026-04-13  
> 适用范围：`release-hardening` 分支全部重构动作  
> 原则：允许改实现，不允许改业务语义、计算口径、权限边界。

## BC-01 接口响应契约

- 契约内容：
  - 所有 API 继续返回 `{ code, data, message }`
  - 成功/失败 code 语义保持现状
- 依据：
  - 架构文档 V1.1 §4.1（line 679-684）
- 回归：
  - 抽样校验 `auth/patients/dialysis/reports` 接口响应字段完整性

## BC-02 角色与权限契约

- 契约内容：
  - 五角色边界不变：`admin/doctor/nurse/head_nurse/qc`
  - 历史兼容：`qc` 与 `quality` 内部等价映射保留
  - 护士“限当班/当日”限制保持不变
- 依据：
  - 需求 V3.0 §2.1（line 90-99）
  - 架构文档 V1.1 §10.1（line 1734-1769）
  - `security-rbac-rules.mdc`
- 回归：
  - 5角色 × 关键接口矩阵（允许/拒绝）自动化抽测

## BC-03 透析充分性公式契约

- 契约内容：
  - Kt/V 使用 Daugirdas II 公式
  - URR 使用 `URR(%) = (1 - R) * 100`
  - 达标条件为 `Kt/V >= 1.2` 且 `URR >= 65`（双条件）
- 依据：
  - 需求 V3.0 §3.2.6（line 278-293）
  - 架构文档 V1.1 §6.3（line 1375-1404）
  - `medical-domain-rules.mdc`
- 回归：
  - `backend/tests/services/ktv-calculator.test.js`
  - 前后端公式样本一致性比对（边界与空值）

## BC-04 超滤警戒契约

- 契约内容：
  - 超滤占比阈值继续使用 `> 5%` 触发警示
- 依据：
  - 需求 V3.0 §3.7（line 595）
  - 架构文档 V1.1 §6.3（line 1395-1402）
- 回归：
  - 透析录入流程边界值回归（4.9/5.0/5.1）

## BC-05 质控上报口径契约

- 契约内容：
  - 五项指标定义与公式保持不变：  
    - 护患比  
    - 体外循环凝血发生率  
    - 体外循环漏血发生率  
    - AVF穿刺损伤发生率  
    - CRBSI发生率（‰）
- 依据：
  - 需求 V3.0 §3.9.1（line 674-724）
  - 架构文档 V1.1 §6.5（line 1432-1494）
  - `hd-qc-reports/SKILL.md`
- 回归：
  - 使用 2026 样例口径做对照回归
  - `reports` 导出内容字段与口径一致性检查

## BC-06 传染病与隔离联动契约

- 契约内容：
  - 传染病筛查周期/提醒规则不变
  - 阳性患者必须使用对应隔离分区设备，排班约束不放宽
- 依据：
  - 需求 V3.0 §3.4.1-3.4.2（line 328-347）
- 回归：
  - 筛查录入后隔离状态切换
  - 排班写入约束与拒绝路径测试

## BC-07 预警规则契约

- 契约内容：
  - 危急值、Kt/V不达标、UF超标、CVC高危等核心规则阈值保持不变
  - 预警级别语义与通知对象不反转
- 依据：
  - 需求 V3.0 §3.7（line 588-603）
  - 架构文档 V1.1 §7.1（line 1501-1572）
- 回归：
  - 规则触发样本回归（单条规则与组合规则）

## BC-08 安全合规契约

- 契约内容：
  - HTTPS、速率限制、JWT 吊销、RBAC 约束继续有效
  - PII 加密存储与响应脱敏继续有效
  - 审计日志不可删（至少满足现有防护语义）
- 依据：
  - 需求 V3.0 §4.2（line 875-883）
  - 架构文档 V1.1 §10.2-10.3（line 1771-1820）
  - `security-rbac-rules.mdc`
- 回归：
  - 安全专项抽检（接口、日志、数据库字段）

## BC-09 定时任务契约

- 契约内容：
  - 核心定时任务仍执行既定业务：预警扫描、月报生成、CQI提醒
  - 时区语义保持 `Asia/Shanghai`
- 依据：
  - 架构文档 V1.1 §7.2（line 1575-1595）
  - 当前代码 `backend/src/jobs/scheduledTasks.js`
- 回归：
  - 预发布任务实跑 + 执行日志校验

## BC-10 数据迁移与恢复契约

- 契约内容：
  - 58 个迁移脚本顺序、幂等性与回滚路径需可验证
  - 备份/恢复流程可达成 RTO/RPO 目标
- 依据：
  - 需求 V3.0 §4.3（line 884-893）
- 回归：
  - “备份 -> 迁移 -> 回滚 -> 恢复”演练记录

## 执行要求

- 任一重构 PR 若触达上述契约，必须在 PR 描述中标注 `BC-*` 编号并附回归证据。
- 任一 `BC-*` 验证失败，禁止进入上线切换步骤。
