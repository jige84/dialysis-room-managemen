# 上线后7天稳定观察计划

> 对应计划步骤：10（上线后稳定观察）  
> 观察周期：上线后 Day 1 ~ Day 7

## 1. 核心观测指标

| 类别 | 指标 | 目标/阈值 |
|---|---|---|
| 可用性 | API 错误率（5xx） | 持续低位，无明显尖峰 |
| 性能 | 关键接口 P95 时延 | 不高于上线前基线 |
| 任务 | 定时任务成功率 | 100% 或可解释失败 |
| 质量 | 报表口径准确率 | 与基线一致 |
| 安全 | 权限异常告警数 | 0 高危越权事件 |

## 2. 每日执行项（Day 1~7）

- [ ] 查看后端错误日志与慢查询趋势
- [ ] 查看关键接口耗时与失败分布
- [ ] 核对定时任务执行结果
- [ ] 抽检报表与核心公式样例
- [ ] 抽检权限拒绝日志（防越权）
- [ ] 记录当日结论与风险项

### 每日固定命令

```bash
cd backend
npm run observe:daily
```

> 产物目录：`docs/qa/release-hardening/generated/post-release-observation/`

## 3. 变更策略（观察期）

- 仅允许低风险修复（不触达不可变业务契约）
- 中高风险改动统一延期到观察期后
- 所有修复必须附最小回归证据

## 4. Day 7 输出物

- [ ] 《上线稳定性总结》
- [ ] 《遗留技术债清单（按风险分级）》
- [ ] 《下一轮重构建议（优先级+投入评估）》
