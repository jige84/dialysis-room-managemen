# 预发布演练与上线切换 Runbook

> 对应计划步骤：9（预发布演练与上线切换）  
> 原则：先备份、再迁移、后切流；每步可回滚。

## 1. 演练前准备

- [ ] 确认分支/版本：`release-hardening` 对应发布 commit
- [ ] 确认不可变契约清单全部通过（`immutable-business-contracts.md`）
- [ ] 准备上线窗口责任人：应用、数据库、运维、业务验收

## 2. 预发布全量演练

### 2.1 构建与门禁

```bash
# frontend
cd frontend
npm run lint
npm run build

# backend
cd ../backend
npm run test:unit
npm run test:rbac-matrix
npm run test:auth-chain
npm run test:smoke-readiness
npm run test:smoke-five-modules
npm run hardening:baseline
npm run audit:migrations
npm run db:drill
# 生产同构环境建议执行（严格备份恢复验收，默认包含“恢复到临时库并校验行数”）
npm run db:acceptance
```

### 2.2 数据演练（备份 -> 迁移 -> 回滚 -> 恢复）

- [ ] 备份数据库并记录备份文件名/时间
- [ ] 执行迁移并记录耗时
- [ ] 执行回滚演练（按既定手册）
- [ ] 恢复到演练前状态，验证数据一致

### 2.3 压测与任务实跑

- [ ] 关键页面压测：透析录入、患者详情、报表导出
- [ ] 定时任务实跑：预警扫描、月报草稿、CQI提醒
- [ ] 记录响应时延与失败率

## 3. 生产上线切换步骤

1. [ ] 冻结发布窗口（禁止非紧急变更）
2. [ ] 执行生产数据库备份（记录备份ID）
3. [ ] 执行数据库迁移
4. [ ] 滚动重启后端服务
5. [ ] 发布前端静态资源
6. [ ] 执行验烟脚本与人工关键流程
7. [ ] 验收通过后解除冻结

## 4. 验烟最小集（上线后立即执行）

- [ ] 登录与鉴权链路（正常/过期/吊销）
- [ ] 透析录入主流程（含医嘱确认）
- [ ] 报表导出（Excel/PDF）
- [ ] 预警中心读取与确认
- [ ] 角色权限抽测（至少 admin/doctor/nurse/qc）

## 5. 回滚策略

### 5.1 代码回滚

- [ ] 切回上一个稳定版本 tag/commit
- [ ] 重启服务并重新验烟

### 5.2 数据恢复

- [ ] 若迁移失败或口径异常，按备份恢复流程恢复
- [ ] 恢复后执行最小验烟并业务确认

### 5.3 降级策略

- [ ] 关闭高风险非核心入口（如可配置 AI 扩展入口）
- [ ] 保留核心透析链路与报表能力
