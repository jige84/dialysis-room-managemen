## 方式二：使用一键更新脚本

### 前提条件
- 已通过 WinSCP 上传了 `update-ecs.sh` 到 `/opt/xuetoushiguanli/`

### 执行更新

#### 方法 A：通过 WinSCP 终端
1. 打开 WinSCP，连接到 ECS
2. 按 `Ctrl+T` 打开终端
3. 执行：
```bash
cd /opt/xuetoushiguanli
chmod +x update-ecs.sh
bash update-ecs.sh
```

#### 方法 B：通过 SSH 客户端
1. 打开 PuTTY 或 Windows Terminal
2. 连接：`ssh root@47.114.111.216`
3. 输入密码
4. 执行：
```bash
bash /opt/xuetoushiguanli/update-ecs.sh
```

### 更新脚本功能

脚本会自动完成以下操作：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 备份当前版本 | 创建压缩包备份 |
| 2 | 备份数据库 | 导出 PostgreSQL 数据 |
| 3 | 拉取最新代码 | 执行 git pull |
| 4 | 更新依赖 | npm install |
| 5 | 数据库迁移 | 执行新的 SQL 迁移 |
| 6 | 重启服务 | PM2 平滑重启 |
| 7 | 验证更新 | 检查 API 响应 |

## 方式三：手动分步更新

如果需要更精细的控制，可以手动执行：

### 1. 备份（重要！）
```bash
# 创建备份目录
mkdir -p /opt/backups

# 备份代码
cd /opt
tar -czf /opt/backups/backup-$(date +%Y%m%d-%H%M%S).tar.gz hemodialysis

# 备份数据库
pg_dump -U hd_app hemodialysis_db > /opt/backups/db-$(date +%Y%m%d-%H%M%S).sql
```

### 2. 更新代码
```bash
cd /opt/hemodialysis

# 如果有 git
git pull origin main

# 或者手动上传新代码后解压
# unzip new-code.zip -d /opt/hemodialysis
```

### 3. 更新后端
```bash
cd /opt/hemodialysis/backend

# 安装依赖
npm install --production

# 执行数据库迁移（如果有）
npm run migrate

# 重启服务
pm2 reload all
```

### 4. 更新前端（如果需要）
```bash
cd /opt/hemodialysis/frontend

# 安装依赖
npm install

# 构建
npm run build
```

### 5. 验证
```bash
# 检查服务状态
pm2 status

# 测试 API
curl http://localhost:3080/api/health

# 查看日志
pm2 logs
```

## 方式四：紧急回滚

如果更新后出现问题，快速回滚：

### 通过 MCP（最简单）
在 Qoder 中输入：
```
回滚血液透析系统到上一个版本
```

### 手动回滚
```bash
# 1. 停止服务
pm2 stop all

# 2. 查看备份列表
ls -lh /opt/backups/backup-before-update-*

# 3. 恢复备份（替换为实际的备份文件名）
cd /opt
rm -rf hemodialysis
tar -xzf /opt/backups/backup-before-update-20260420-120000.tar.gz

# 4. 恢复数据库（如果需要）
psql -U hd_app hemodialysis_db < /opt/backups/db-backup-20260420-120000.sql

# 5. 重启服务
pm2 restart all
```

## 常见问题

### Q: 更新后服务无法启动？
**A**: 
1. 查看错误日志：`pm2 logs`
2. 检查环境变量：`cat /opt/hemodialysis/backend/.env`
3. 检查端口占用：`netstat -tlnp | grep 3080`
4. 尝试回滚到上一个版本

### Q: 数据库迁移失败？
**A**:
1. 查看具体错误信息
2. 手动执行迁移文件：`psql -U hd_app hemodialysis_db < migrations/xxx.sql`
3. 或者跳过迁移，联系开发团队

### Q: 前端页面显示空白？
**A**:
1. 检查前端构建：`ls /opt/hemodialysis/frontend/dist`
2. 重新构建前端：`cd frontend && npm run build`
3. 检查 Nginx 配置：`nginx -t`

### Q: 如何查看更新历史？
**A**:
```bash
# 查看 git 提交历史
cd /opt/hemodialysis
git log --oneline -20

# 查看更新日志
cat /var/log/hemodialysis-update.log
```

## 推荐的更新流程

### 日常小更新（每周）
1. 使用 MCP 指令："更新系统"
2. 等待自动完成
3. 验证功能正常

### 版本大更新（每月）
1. 提前在测试环境验证
2. 选择低峰时段
3. 手动执行更新脚本
4. 全面测试各项功能
5. 监控 24 小时

### 紧急修复（随时）
1. 快速备份当前状态
2. 应用紧急修复
3. 验证修复有效
4. 详细记录变更
